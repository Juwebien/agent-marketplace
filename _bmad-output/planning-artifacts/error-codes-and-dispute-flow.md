## Section 1: Standardized API Error Codes

All API responses for errors follow this JSON format:
```json
{
  "error": true,
  "code": "MODULE_CODE",  // e.g., "AUTH_001"
  "message": "Human-readable error message",
  "details": {}  // Optional object with additional context (e.g., { field: "email", reason: "invalid" })
}
```

Error codes are prefixed by module (e.g., `AUTH_`, `MISSIONS_`) and numbered sequentially. This list is exhaustive for the specified modules, covering common failure scenarios in an AI agent marketplace.

## Section 2: Fiat/Stripe Dispute Flow (Chargeback Handling)

### Flow Description

This flow handles a chargeback (dispute) initiated by a client via Stripe for fiat payments in the AI agent marketplace. The process ensures compliance, freezes escrowed funds (e.g., in a Web3 escrow smart contract), notifies the provider (e.g., the agent owner or mission creator), and escalates to multisig arbitration if needed.

**Assumptions**:
- Backend uses Node.js with Express.js for API.
- Database: PostgreSQL (or similar) with tables like `payments`, `escrows`, `notifications`.
- Web3 integration: Ethereum-based escrow contract (e.g., via ethers.js).
- Multisig: A Gnosis Safe or similar for arbitration (e.g., 2/3 multisig with admins).
- Stripe webhooks are configured to send `charge.dispute.created` events.

**Step-by-Step Flow**:
1. **Stripe Webhook Trigger**: Stripe sends a webhook to `/webhooks/stripe` endpoint on chargeback creation (`charge.dispute.created` event).
2. **API Handler**: Validate webhook signature, parse event, and retrieve related payment/mission data from DB.
3. **DB Update**: Mark the payment as disputed in DB, log the event.
4. **Escrow Freeze**: Call Web3 escrow contract to freeze funds associated with the mission/payment (prevent release to provider).
5. **Notification to Provider**: Send email/push notification to the provider (agent owner) via a service like SendGrid or Firebase.
6. **Arbitration Multisig**: If unresolved (e.g., after 7 days), escalate to multisig for manual review/release/refund. (This is triggered via a cron job or manual admin action, not in webhook.)

The flow ensures atomicity where possible (e.g., using DB transactions).

### TypeScript Code for Handlers

Below is ready-to-commit TypeScript code for the webhook handler and related functions. Assumes dependencies: `express`, `stripe`, `ethers`, `pg` (for DB), and a notification service (e.g., SendGrid).

```typescript
// src/handlers/stripeWebhook.ts
import express, { Request, Response } from 'express';
import Stripe from 'stripe';
import { ethers } from 'ethers';
import { Pool } from 'pg'; // Assuming PostgreSQL pool
import sgMail from '@sendgrid/mail'; // For notifications

// Config (load from env)
const STRIPE_SECRET_KEY = process.env.STRIPE_SECRET_KEY!;
const STRIPE_WEBHOOK_SECRET = process.env.STRIPE_WEBHOOK_SECRET!;
const ETH_PROVIDER_URL = process.env.ETH_PROVIDER_URL!;
const ESCROW_CONTRACT_ADDRESS = process.env.ESCROW_CONTRACT_ADDRESS!;
const MULTISIG_ADDRESS = process.env.MULTISIG_ADDRESS!; // For logging/escalation
const SENDGRID_API_KEY = process.env.SENDGRID_API_KEY!;

sgMail.setApiKey(SENDGRID_API_KEY);

const stripe = new Stripe(STRIPE_SECRET_KEY, { apiVersion: '2023-10-16' });
const dbPool = new Pool({ connectionString: process.env.DATABASE_URL });
const ethProvider = new ethers.providers.JsonRpcProvider(ETH_PROVIDER_URL);
// Assume Escrow ABI is imported or defined here
const escrowABI = [ /* ABI for freezeEscrow function, e.g., "function freeze(uint256 escrowId)" */ ];
const escrowContract = new ethers.Contract(ESCROW_CONTRACT_ADDRESS, escrowABI, ethProvider.getSigner());

// Webhook endpoint
const router = express.Router();

router.post('/webhooks/stripe', express.raw({ type: 'application/json' }), async (req: Request, res: Response) => {
  const sig = req.headers['stripe-signature'] as string;

  let event: Stripe.Event;
  try {
    event = stripe.webhooks.constructEvent(req.body, sig, STRIPE_WEBHOOK_SECRET);
  } catch (err) {
    return res.status(400).json({ error: true, code: 'PAYMENTS_009', message: 'Webhook signature verification failed', details: { error: (err as Error).message } });
  }

  if (event.type === 'charge.dispute.created') {
    const dispute = event.data.object as Stripe.Dispute;
    const chargeId = dispute.charge as string; // Assuming charge ID links to payment

    try {
      // Step 2: Retrieve from DB
      const client = await dbPool.connect();
      const paymentRes = await client.query('SELECT * FROM payments WHERE stripe_charge_id = $1', [chargeId]);
      if (paymentRes.rows.length === 0) {
        client.release();
        return res.status(404).json({ error: true, code: 'PAYMENTS_001', message: 'Payment not found' });
      }
      const payment = paymentRes.rows[0];
      const missionId = payment.mission_id; // Assume linked to mission
      const escrowId = payment.escrow_id; // Web3 escrow ID
      const providerId = payment.provider_id; // Agent owner ID

      // Step 3: Update DB (transaction for atomicity)
      await client.query('BEGIN');
      await client.query('UPDATE payments SET status = $1, updated_at = NOW() WHERE id = $2', ['disputed', payment.id]);
      await client.query('INSERT INTO disputes (payment_id, stripe_dispute_id, status) VALUES ($1, $2, $3)', [payment.id, dispute.id, 'pending']);
      await client.query('COMMIT');
      client.release();

      // Step 4: Freeze Escrow
      const tx = await escrowContract.freeze(escrowId);
      await tx.wait(); // Wait for confirmation

      // Step 5: Notify Provider
      const providerEmailRes = await dbPool.query('SELECT email FROM users WHERE id = $1', [providerId]);
      const providerEmail = providerEmailRes.rows[0]?.email;
      if (providerEmail) {
        await sgMail.send({
          to: providerEmail,
          from: 'no-reply@ai-marketplace.com',
          subject: 'Chargeback Dispute Notification',
          text: `A chargeback has been filed for mission ${missionId}. Escrow frozen. Review at: https://dashboard.ai-marketplace.com/disputes/${dispute.id}`,
        });
      }

      // Step 6: Log for Arbitration (escalation handled separately, e.g., via cron)
      console.log(`Dispute ${dispute.id} escalated to multisig at ${MULTISIG_ADDRESS} for review.`);

      return res.status(200).json({ received: true });
    } catch (err) {
      console.error(err);
      return res.status(500).json({ error: true, code: 'PAYMENTS_003', message: 'Transaction failed', details: { error: (err as Error).message } });
    }
  }

  // Handle other events if needed
  return res.status(200).json({ received: true });
});

export default router;

// Usage: app.use(router); in main app.ts
```
